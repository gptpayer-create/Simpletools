UUID v1
Time-based + MAC address. Monotonically increasing but leaks machine info.
UUID v4
Randomly generated. Most widely used. 122 bits of randomness.
UUID v5
Deterministic. SHA-1 hash of a namespace + name. Same input = same UUID.
ULID
Universally Unique Lexicographically Sortable Identifier. Timestamp prefix.
NanoID
Compact, URL-safe random ID. Shorter than UUID, configurable length.
ObjectID
MongoDB-compatible 12-byte identifier: timestamp + random + counter.

Frequently Asked Questions

What is a UUID?
A UUID (Universally Unique Identifier) is a 128-bit identifier guaranteed to be unique across all systems.
What is the difference between UUID v1 and v4?
UUID v1 is based on time and machine ID. UUID v4 is fully random. v4 is recommended for most use cases.
Are generated UUIDs truly unique?
UUID v4 has 2^122 possible values, making collisions practically impossible.
